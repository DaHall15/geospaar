1 Overview

In this unit we finally begin to start working with spatial data. We will learn to use R to do the types of analyses you typically associate with traditional GIS software, such as Idrisi/Terrset, ArcGIS, QGIS, and friends. We will also continue to use and build on the basic R programming and reproducibility skills you learned in Unit 1.

Three assignments will be given during this unit. You should also finalize the subject of your final project before the mid-point of this unit, and begin to assemble the data you need to undertake the project, and outline the approach you plan to use. The more effort you put into planning (and even beginning the analysis for) your project, the more R skills you will acquire. Why? Because you will be be thinking about how to make R produce solutions for problems you are interested in solving, which requires more exploration and creativity than simply tackling assignments that I give you. Also, since each of you will be doing different projects, there is more potential for peer-to-peer learning, as each of you will learn about and share different aspects of R.

The learning goals for this unit are as follows - by the end of this unit, you should:

  • Understand/know: the strengths and limitations of working with spatial data in R; possible workarounds for these limitations; the differences between the primary spatial object classes; the three external libraries (GDAL, GEOS, and PROJ.4) that R (and many other GIS) uses to provide spatial capabilities; the fact that R’s current sp-based classes and methods are in the process of being replaced by simple features (via the new sf package)

  • Be able to do the following: perform basic vector operations (intersections, unions, differences) and manipulations (merges/joins); raster operations (algebra, focal analyses, terrain characteristics) and manipulations (aggregate/disaggregate, resample, reproject); convert rasters to vectors and vice versa; visualize spatial data; extract quantities/statistics from spatial data; run simple spatial models and map their predictions; get help on R-sig-geo (i.e. know how to create a reproducible examples)

2.1 Class 12 (Oct 9). Spatial data classes

  • Required reading:
    • Read the scanned pdf of Chapter 3 from Applied Spatial Data Analysis with R, which you can get by download from this link (please let me know if this doesn’t work). This is primarily a reading assignment (but see next bullet point)–focus on learning about the different types of spatial objects and methods, beginning with section 3, and particularly regarding the different ways that spatial objects are structured, building in complexity from the simplest Spatial class up through SpatialPoints*, then SpatialLines*, and then SpatialPolygons* for vectors, and how data.frames are associated with each. For rasters, understand the differences between SpatialGrids*, SpatialPixels*, and raster classes. You also want to pay attention to key “slots” in these various S4 classes (e.g. “bbox”, “proj4string”, “coords”, “data”) and the methods that are used to query slots (e.g. bbox, proj4string, coordinates, slot [note this list of methods is not a 1:1 map to slots]).

    • There is code and data associated with this chapter. Because there is a lot of it, you are not expected to work through it all. However, you are expected to try at least some of it, particularly where you are building up simple spatial objects (up through SpatialPoints*, so 2.3-2.4), and in the section (2.8) on the raster package, where you have to read in and query basic properties of the raster (up until the code block containing the writeStart function in it on page 55–which you can of course do if you are interested). For speed, it is always easiest to copy, paste, and adapt code. However, that is tough to do out of the pdf.
